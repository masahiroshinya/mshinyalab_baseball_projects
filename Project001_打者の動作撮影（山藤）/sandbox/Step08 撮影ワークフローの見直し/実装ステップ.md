# 同期用LEDの推奨仕様

## なぜLEDで同期をとるのか

OSMO Action 3 は 240fps 撮影時にタイムコード同期機能が使えない（60fps以上では無効）。
そのため、全カメラの画角に映る**共通の視覚イベント（LED点灯）**を利用してソフトウェア的にフレーム同期をとる。

## LED の選定基準

| 項目 | 推奨仕様 | 理由 |
|:---|:---|:---|
| **色** | **白色** または **暖白色**（3000〜6500K） | カラーフィルタの影響を受けにくく、グレースケール変換後の輝度コントラストが最大になる |
| **明るさ** | **100〜300ルーメン程度**（懐中電灯クラス） | 屋外でも映像上で明確な輝度変化が得られる。暗すぎると背景光に埋もれる |
| **大きさ（発光面）** | **直径 1〜3 cm** 程度の点光源 | 映像上で 10〜30 ピクセル程度のROIに収まるサイズ。大きすぎると輝度変化がなだらかになる |
| **スイッチ方式** | **押しボタン式**（押している間だけ点灯） | ランダム間隔での複数回点灯が容易。トグル式は操作ミスが起きやすい |
| **応答速度** | **即時点灯**（LED固有の特性で問題なし） | 白熱球やフィラメント式は立ち上がり遅延があるため不可 |
| **電源** | 電池式（単三/単四 または USB充電式） | 撮影現場での取り回しの良さ |

## 具体的な製品例

| 分類 | 製品例 | 概算価格 | 備考 |
|:---|:---|:---|:---|
| **最も手軽** | スマホのフラッシュライト | 0円 | 一番簡単。ただし明るさ調整ができない場合がある |
| **推奨** | 小型LEDペンライト（100〜200ルーメン） | 500〜2,000円 | ボタン押下で即時点灯、持ちやすい。100均でも可 |
| **高精度向け** | USB接続LEDボタン（Arduino + LED） | 1,000〜3,000円 | 将来的に自動化する場合の拡張性がある |

## 撮影時の注意事項

1. **カメラの露出は手動（固定）に設定する** — 自動露出だとLED点灯時にカメラが露出を補正し、輝度変化が平滑化されてしまう
2. **LEDは全カメラの画角内に映る位置に置く** — 各カメラから見えるならば、LEDの向きは厳密でなくてよい
3. **複数回・ランダム間隔で点灯する** — 等間隔だとクロスコリレーションに複数ピークが生じ、誤同期の原因になる
4. **1回あたりの点灯時間は 0.3〜1秒**程度 — 短すぎると240fpsでも数フレームしか映らず不安定。長すぎるとエッジがぼやける
5. **最低3〜5回は点灯させる** — 回数が多いほどクロスコリレーションのSN比が向上する

動画ファイルのパスを入力し、フレーム数・FPS・解像度などの基本情報を取得する。

```python
# ============================================
# 動画情報取得の関数定義
# ============================================
def get_video_info(path):
    """動画ファイルの基本情報を取得する"""
    cap = cv2.VideoCapture(path)
    if not cap.isOpened():
        print(f"Error: Could not open video file {path}")
        return None

    info = {
        "path": path,
        "fps": cap.get(cv2.CAP_PROP_FPS),
        "width": int(cap.get(cv2.CAP_PROP_FRAME_WIDTH)),
        "height": int(cap.get(cv2.CAP_PROP_FRAME_HEIGHT)),
        "n_frames": int(cap.get(cv2.CAP_PROP_FRAME_COUNT)),
    }
    info["duration_sec"] = info["n_frames"] / info["fps"] if info["fps"] > 0 else 0
    cap.release()
    return info

# ============================================
# 動画ファイルの選択
# ============================================
#@title 動画ファイルの選択 { display-mode: "form" }
camera1_path = "/content/cam1.MP4" #@param {type:"string"}
camera2_path = "/content/cam2.MP4" #@param {type:"string"}

video_paths = [camera1_path, camera2_path]
video_infos = []
for i, path in enumerate(video_paths):
    info = get_video_info(path)
    if info is None:
        raise FileNotFoundError(f"Camera {i+1} の動画ファイルが見つかりません: {path}")
    video_infos.append(info)
    print(f"Camera {i+1}: FPS={info['fps']}, {info['width']}x{info['height']}, "
          f"{info['n_frames']}frames, {info['duration_sec']:.1f}sec")

nCamera = len(video_paths)
print(f"\nカメラ台数: {nCamera}")
```

## 手順1-4：LED点灯フレームの探索

スライダーでフレームを動かし、LEDが点灯しているフレームを見つける。
LED が光っているフレームを使うと、LEDの位置がはっきり分かるため、次のROI指定が楽になる。

> **LEDは1フレーム目に点灯している必要はありません。**
> 撮影中のどこかで点灯していれば、スライダーで探して見つけられます。

```python
# ============================================
# フレームブラウザ（Camera 1 のみ、LEDを探す用）
# ============================================
def read_frame(path, frame_no):
    """動画の指定フレームを読み込む"""
    cap = cv2.VideoCapture(path)
    cap.set(cv2.CAP_PROP_POS_FRAMES, frame_no)
    ret, frame = cap.read()
    cap.release()
    if not ret:
        raise RuntimeError(f"フレーム読み込み失敗: {path}, frame={frame_no}")
    return cv2.cvtColor(frame, cv2.COLOR_BGR2RGB)

# --- Camera 1 をベースにLED点灯フレームを探す ---
n_frames_cam1 = video_infos[0]["n_frames"]
scale = 0.4  # 表示倍率（画像が大きすぎる場合に縮小）

    value=0, min=0, max=n_frames_cam1 - 1,
    step=1, description='Frame:',
    layout=widgets.Layout(width='80%')
)
btn_prev = widgets.Button(description='◀ -1')
btn_next = widgets.Button(description='+1 ▶')
btn_prev10 = widgets.Button(description='◀◀ -10')
btn_next10 = widgets.Button(description='+10 ▶▶')
btn_prev100 = widgets.Button(description='◀◀◀ -100')
btn_next100 = widgets.Button(description='+100 ▶▶▶')
output_area = widgets.Output()

def step(delta):
    frame_slider.value = max(0, min(n_frames_cam1 - 1, frame_slider.value + delta))

btn_prev.on_click(lambda _: step(-1))
btn_next.on_click(lambda _: step(1))
btn_prev10.on_click(lambda _: step(-10))
btn_next10.on_click(lambda _: step(10))
btn_prev100.on_click(lambda _: step(-100))
btn_next100.on_click(lambda _: step(100))

def show_frame(change):

        ax.imshow(small)
```

## 手順1-5：LED ROIの指定（スライダーで赤い□を操作）

前のステップで見つけたLED点灯フレームの画像を使い、**x / y のスライダーを動かして赤い□（ROI）をLEDの位置に合わせる**。座標の手入力は不要。

> **ROI（関心領域）とは？**
> 画像中のLED周辺を囲む小さな四角形のこと。この□の中の輝度変化だけを追跡して、同期のタイミングを計算する。

```python
# ============================================
# LED ROI の指定（スライダー操作式）
# ============================================
# カメラごとに、LED点灯フレームの画像上でROIを指定する。
# x, y のスライダーを動かすと、赤い□がリアルタイムに移動する。

#@title LED点灯フレーム番号の入力 { display-mode: "form" }
#@markdown 手順1-4で見つけたLED点灯フレーム番号を入力してください。
led_frame_no = 100 #@param {type:"integer"}
roi_half_size = 30 #@param {type:"integer"}

# LED点灯フレームを全カメラで読み込み（ROI指定で使う）
led_frames = [read_frame(p, led_frame_no) for p in video_paths]
roi_centers = []  # ここに結果が入る

for cam_idx in range(nCamera):
    frame = led_frames[cam_idx]
    h, w = frame.shape[:2]

    print(f"\n{'='*50}")
    print(f"Camera {cam_idx+1}: スライダーでLEDの位置に赤い□を合わせてください")
    print(f"{'='*50}")

    # スライダー
    x_slider = widgets.IntSlider(
        value=w // 2, min=0, max=w - 1,
        step=1, description='X位置:',
        layout=widgets.Layout(width='80%'),
        style={'description_width': '60px'}
    )
    y_slider = widgets.IntSlider(
        value=h // 2, min=0, max=h - 1,
        step=1, description='Y位置:',
        layout=widgets.Layout(width='80%'),
        style={'description_width': '60px'}
    )
    size_slider = widgets.IntSlider(
        value=roi_half_size, min=10, max=100,
        step=5, description='ROIサイズ:',
        layout=widgets.Layout(width='50%'),
        style={'description_width': '80px'}
    )
    confirm_btn = widgets.Button(
        description=f'✅ Camera {cam_idx+1} のROIを確定',
        button_style='success',
        layout=widgets.Layout(width='300px')
    )
    status_label = widgets.HTML(value="")
    roi_output = widgets.Output()

    # 状態管理
    confirmed = [False]
    saved_roi = [None]

    def make_update(frm, x_sl, y_sl, sz_sl, out, cam_i):
        def update(change=None):
            cx, cy, sz = x_sl.value, y_sl.value, sz_sl.value
            with out:
                clear_output(wait=True)
                fig, ax = plt.subplots(1, 1, figsize=(10, 7))
                ax.imshow(frm)
                # ROI矩形
                rect = plt.Rectangle(
                    (cx - sz, cy - sz), sz * 2, sz * 2,
                    linewidth=2, edgecolor='red', facecolor='red', alpha=0.15
                )
                ax.add_patch(rect)
                # 中心の十字
                ax.plot(cx, cy, 'r+', markersize=25, markeredgewidth=3)
                # ROI枠線（強調）
                rect2 = plt.Rectangle(
                    (cx - sz, cy - sz), sz * 2, sz * 2,
                    linewidth=3, edgecolor='red', facecolor='none'
                )
                ax.add_patch(rect2)
                ax.set_title(f"Camera {cam_i+1} — ROI中心: ({cx}, {cy}), サイズ: ±{sz}px\n"
                             f"赤い□にLEDが入るようにスライダーを調整",
                             fontsize=12)
                ax.axis('off')
                plt.tight_layout()
                plt.show()
        return update

    update_fn = make_update(frame, x_slider, y_slider, size_slider, roi_output, cam_idx)

    x_slider.observe(update_fn, names='value')
    y_slider.observe(update_fn, names='value')
    size_slider.observe(update_fn, names='value')

    def make_confirm(x_sl, y_sl, sz_sl, conf, saved, label, cam_i):
        def on_confirm(btn):
            saved[0] = {"x": x_sl.value, "y": y_sl.value, "roi_size": sz_sl.value}
            conf[0] = True
            label.value = (f"<b style='color:green'>✅ Camera {cam_i+1} 確定: "
                           f"({x_sl.value}, {y_sl.value}), サイズ ±{sz_sl.value}px</b>")
        return on_confirm

    confirm_btn.on_click(make_confirm(x_slider, y_slider, size_slider,
                                       confirmed, saved_roi, status_label, cam_idx))

    display(x_slider, y_slider, size_slider, confirm_btn, status_label, roi_output)
    update_fn()  # 初期表示

    # 結果は confirm_btn クリック後に saved_roi[0] に入る
    # 次のセルで roi_centers に集約する

print("\n" + "="*50)
print("全カメラのスライダーを調整し、✅ボタンを押してください。")
print("確定したら、次のセルに進んでください。")
```

**操作方法：**
1. **X位置スライダー**: 左右に動かして赤い□を水平方向にLEDの位置へ移動
2. **Y位置スライダー**: 左右に動かして赤い□を垂直方向にLEDの位置へ移動
3. **ROIサイズスライダー**: □の大きさを調整（通常は30で十分）
4. **赤い□がLEDを囲んだら「✅ 確定」ボタン**をクリック
5. 全カメラで繰り返す

> **うまくいかない場合:**
> LEDが小さすぎて見つからない場合は、手順1-4のスライダーで別の点灯フレームを探してください。

## 手順1-6：ROI内の輝度時系列の抽出

指定したROI内のピクセル輝度の平均値を全フレームにわたって取得する。動画が長い場合、この処理には時間がかかる。

```python
# ============================================
# ROI 輝度時系列の抽出関数
# ============================================
def extract_roi_brightness(video_path, roi_center, roi_size=30):
    """
    動画のROI内の平均輝度を全フレームにわたって取得する。

    Args:
        video_path: 動画ファイルのパス
        roi_center: dict with "x", "y" keys
        roi_size: ROIの半径

    Returns:
        brightness: numpy array of mean brightness per frame
    """
    cap = cv2.VideoCapture(video_path)
    n_frames = int(cap.get(cv2.CAP_PROP_FRAME_COUNT))
    width = int(cap.get(cv2.CAP_PROP_FRAME_WIDTH))
    height = int(cap.get(cv2.CAP_PROP_FRAME_HEIGHT))

    cx, cy = roi_center["x"], roi_center["y"]
    x1 = max(0, cx - roi_size)
    y1 = max(0, cy - roi_size)
    x2 = min(width, cx + roi_size)
    y2 = min(height, cy + roi_size)

    brightness = np.zeros(n_frames)

    # 進捗表示付きで全フレーム走査
    report_interval = max(1, n_frames // 10)  # 10%ごとに表示
    for f in range(n_frames):
        ret, frame = cap.read()
        if not ret:
            break
        gray = cv2.cvtColor(frame, cv2.COLOR_BGR2GRAY)
        roi = gray[y1:y2, x1:x2]
        brightness[f] = np.mean(roi)

        if (f + 1) % report_interval == 0:
            pct = (f + 1) / n_frames * 100
            print(f"  進捗: {pct:.0f}% ({f+1}/{n_frames} frames)")

    cap.release()
    print(f"  ✅ 輝度抽出完了: {n_frames} frames, ROI=({x1},{y1})-({x2},{y2})")
    return brightness

# ============================================
# 全カメラの輝度時系列を取得
# ============================================
brightness_list = []
for i in range(nCamera):
    print(f"\nCamera {i+1} の輝度を抽出中...")
    b = extract_roi_brightness(
        video_paths[i],
        roi_centers[i],
        roi_centers[i]["roi_size"]
    )
    brightness_list.append(b)

print(f"\n✅ 全カメラの輝度抽出が完了しました")
```

**確認ポイント：**
- 進捗が10%ごとに表示され、100%まで到達すること
- エラーなく全カメラの抽出が完了すること
- ⏱ 処理時間の目安: 240fps × 30秒 = 7200フレーム → 約1〜3分（Colabの性能に依存）

## 手順1-8：微分信号の生成とクロスコリレーションによるオフセット計算

輝度時系列を微分してエッジ信号に変換し、カメラ1を基準にクロスコリレーションでフレームオフセットを計算する。

```python
# ============================================
# 微分信号の生成 + クロスコリレーション
# ============================================
def compute_offset_by_cross_correlation(brightness_ref, brightness_target):
    """
    微分信号のクロスコリレーションにより、フレームオフセットを計算する。

    Args:
        brightness_ref: 基準カメラの輝度時系列（numpy array）
        brightness_target: 対象カメラの輝度時系列（numpy array）

    Returns:
        offset: フレーム単位のオフセット（正: targetが遅れている、負: targetが進んでいる）
        correlation: クロスコリレーション結果
        lags: ラグの配列
        diff_ref: 基準カメラの微分信号
        diff_target: 対象カメラの微分信号
    """
    # 微分信号（エッジ検出）
    diff_ref = np.diff(brightness_ref)
    diff_target = np.diff(brightness_target)

    # 正規化
    diff_ref_norm = (diff_ref - np.mean(diff_ref)) / (np.std(diff_ref) + 1e-8)
    diff_target_norm = (diff_target - np.mean(diff_target)) / (np.std(diff_target) + 1e-8)

    # クロスコリレーション
    correlation = signal.correlate(diff_ref_norm, diff_target_norm, mode='full')
    lags = signal.correlation_lags(len(diff_ref_norm), len(diff_target_norm), mode='full')

    # 最大相関のラグ = オフセット
    offset = lags[np.argmax(correlation)]
    peak_corr = np.max(correlation)

    return offset, correlation, lags, diff_ref, diff_target, peak_corr
```

### 手順2-2b：画像を横並びに結合する関数の作成
「横並び表示」には、数値計算ライブラリの **NumPy (`np.hstack`)** を用いています。画像はコンピュータにとって「ピクセルの配列（行列）」であるため、複数カメラの配列同士を横に繋げる（Horizontal Stack）ことで実現しています。

```python
# ============================================
# 横並び表示（結合）処理
# ============================================
scale = 0.3 # 処理を軽くするための縮小率
selected_frames = [] # ユーザーが選択したフレーム番号を保存するリスト

# スライダーを動かすたびに開閉すると処理落ちするため、動画をあらかじめ開いて保持します
synced_caps = [cv2.VideoCapture(p) for p in synced_paths]

def get_combined_frame(caps, frame_no, scale):
    """指定したフレーム番号の画像を全カメラ分読み込み、横並びに結合して返す関数"""
    frames = []
    
    for cap in caps:
        cap.set(cv2.CAP_PROP_POS_FRAMES, frame_no)
        ret, frame = cap.read()
        
        if ret:
            # OpenCVを用いてサイズを縮小し、色空間の補正（BGRからRGBへ）を行う
            h, w = frame.shape[:2]
            small = cv2.resize(frame, (int(w * scale), int(h * scale)))
            frames.append(cv2.cvtColor(small, cv2.COLOR_BGR2RGB))
            
    if frames:
        # NumPyのhstack（Horizontal Stack）を用いて、複数の画像を横並びにピッタリ結合する
        combined = np.hstack(frames)
        return combined
    
    return None

print("✅ 画像の横並び結合関数の準備が完了しました。")
```

## 手順2-3：コーナー検出と検証

選択したフレームに対して全カメラでチェスボードのコーナー検出を実行し、失敗したフレームを警告する。

```python
def detect_corners_for_frames(video_paths, frame_numbers, chess_size):
    """
    指定フレームに対してコーナー検出を実行する。

    Args:
        video_paths: 動画パスのリスト
        frame_numbers: フレーム番号のリスト
        chess_size: (cols, rows) チェスボードの内部コーナー数

    Returns:
        results: dict {frame_no: {"success": [bool, ...], "corners": [corners, ...]}}
    """
    nCamera = len(video_paths)
    criteria = (cv2.TERM_CRITERIA_EPS + cv2.TERM_CRITERIA_MAX_ITER, 30, 0.001)
    results = {}

    for frame_no in frame_numbers:
        cam_results = {"success": [], "corners": []}

        for i, path in enumerate(video_paths):
            cap = cv2.VideoCapture(path)
            cap.set(cv2.CAP_PROP_POS_FRAMES, frame_no)
            ret, frame = cap.read()
            cap.release()

            if not ret:
                cam_results["success"].append(False)
                cam_results["corners"].append(None)
                continue

            gray = cv2.cvtColor(frame, cv2.COLOR_BGR2GRAY)
            found, corners = cv2.findChessboardCorners(gray, chess_size, None)

            if found:
                corners_refined = cv2.cornerSubPix(
                    gray, corners, (11, 11), (-1, -1), criteria
                )
                cam_results["success"].append(True)
                cam_results["corners"].append(corners_refined)
            else:
                cam_results["success"].append(False)
                cam_results["corners"].append(None)

        results[frame_no] = cam_results

        # 結果表示
        status_icons = ["✅" if s else "❌" for s in cam_results["success"]]
        all_ok = all(cam_results["success"])
        prefix = "  " if all_ok else "⚠ "
        cam_status = " | ".join([f"Cam{j+1}:{icon}" for j, icon in enumerate(status_icons)])
        print(f"{prefix}Frame {frame_no:>6d}: {cam_status}")

    return results

# コーナー検出を実行
chess_size = (chess_cols, chess_rows)
# selected_frames は手順2-2で選択・蓄積したリストがそのまま使用されます

print(f"=== コーナー検出 ({len(selected_frames)} フレームに対して実行) ===\n")

# 全カメラで成功したフレームのみ抽出
valid_frames = [
    fn for fn, r in corner_results.items()
    if all(r["success"])
]
failed_frames = [
    fn for fn, r in corner_results.items()
    if not all(r["success"])
]

print(f"\n--- 結果 ---")
print(f"全カメラで検出成功: {len(valid_frames)} フレーム")
print(f"一部失敗:         {len(failed_frames)} フレーム → {failed_frames}")

if len(valid_frames) < nRequiredFrames:
    print(f"\n⚠ 有効フレーム数が {nRequiredFrames} 枚に達していません。"
          f"フレームブラウザに戻り、追加のフレームを選択してください。")
```
