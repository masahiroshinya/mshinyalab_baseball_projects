# Step 07 実装ステップ：キャリブレーションの「犯人探し」と精度向上

このドキュメントは、Step06のノートブック内に組み込んで、キャリブレーション全体の精度（RMS）を悪化させている「悪い画像（コーナーがズレている、歪みが酷い等）」を特定し、それらを除外して再計算するための具体的な手順です。

## 作業の概要と工数
- **想定作業時間**: 約15分 〜 30分
- **作業の重さ**: 非常に軽い（提供されたコードをColabにコピペし、実行結果や画像の目視確認を行う作業がメインです）

**全体の流れ（5ステップ）**:
1. **誤差計算関数の追加**: 各画像の再投影誤差を計算する関数を追加する。
2. **悪い画像の特定**: 誤差が閾値（例: 1.0px）を超える画像を特定し、インデックスを抽出する。
3. **目視での原因確認（推奨）**: 抽出された画像をプロットし、誤差の原因（ブレ、歪み等）を目視で確認する。
4. **データのクリーンナップ**: 不良画像を配列から除外し、カメラーペアを崩さずにクリーンな配列を再構築する。
5. **キャリブレーションの再実行**: クリーンな配列を用いて再計算し、RMSの大幅な改善（0.5px程度）を確認する。

---

## 1. ノートブックへの関数追加

まずは、キャリブレーション実行直後のセルに、各画像ごとの「再投影誤差（RMS）」を算出する関数を定義します。

```python
import numpy as np
import cv2

def analyze_reprojection_errors(objpoints, imgpoints, rvecs, tvecs, K, D):
    """
    各画像に対する再投影誤差を計算し、配列で返す関数。
    """
    total_error = 0
    errors = []
    
    for i in range(len(objpoints)):
        # 3Dのオブジェクトポイントを、計算した内部パラメータで2D画像上に再投影する
        imgpoints2, _ = cv2.projectPoints(objpoints[i], rvecs[i], tvecs[i], K, D)
        
        # 実際の検出点と再投影された点のユークリッド距離（誤差）を求める
        error = cv2.norm(imgpoints[i], imgpoints2, cv2.NORM_L2) / len(imgpoints2)
        errors.append(error)
        total_error += error
        
        print(f"画像インデックス {i:02d}: 誤差 = {error:.4f} px")
        
    mean_error = total_error / len(objpoints)
    print(f"\n---> 全体の平均誤差: {mean_error:.4f} px")
    
    return errors
```

## 2. 誤差の算出と「悪い画像」の特定

`cv2.calibrateCamera()` を実行してパラメータ（`ret1, K1, D1, rvecs1, tvecs1` 等）を取得した直後のセルで、先ほどの関数を実行して犯人を探します。
ここでは、目安として**「誤差が 1.0 ピクセル以上」**の画像を不良データと見なします。

```python
print("=== Camera 1 の誤差分析 ===")
errors1 = analyze_reprojection_errors(objpoints_cam1, imgpoints_cam1, rvecs1, tvecs1, K1, D1)

# 誤差が大きすぎる画像のインデックスを特定
THRESHOLD = 1.0
bad_indices1 = [i for i, err in enumerate(errors1) if err > THRESHOLD]
print(f"Cam1 の除外推奨インデックス: {bad_indices1}")

print("\n=== Camera 2 の誤差分析 ===")
errors2 = analyze_reprojection_errors(objpoints_cam2, imgpoints_cam2, rvecs2, tvecs2, K2, D2)
bad_indices2 = [i for i, err in enumerate(errors2) if err > THRESHOLD]
print(f"Cam2 の除外推奨インデックス: {bad_indices2}")
```

## 3. なぜ悪かったのか？目視での原因確認（推奨）

除外する前に、なぜその画像の誤差が大きいのか（ブレているのか、チェスボードが端すぎて歪んでいるのか）を必ず目視で確認し、今後の撮影に活かします。

```python
import matplotlib.pyplot as plt

# 例: Camera 1 の悪い画像（最初の1枚）を表示してみる
if len(bad_indices1) > 0:
    bad_idx = bad_indices1[0]
    # images_cam1 に元画像のパス一覧が入っていると想定
    bad_img_path = images_cam1[bad_idx] 
    
    img = cv2.imread(bad_img_path)
    img_rgb = cv2.cvtColor(img, cv2.COLOR_BGR2RGB)
    
    plt.figure(figsize=(8, 6))
    plt.imshow(img_rgb)
    plt.title(f"Bad Image (Cam1 Index: {bad_idx}) - Error: {errors1[bad_idx]:.2f}px")
    plt.show()
```

## 4. 悪い画像を除外したデータ配列の再作成

特定した `bad_indices` を使って、`objpoints` と `imgpoints` から悪い要素を取り除いた「クリーンなリスト」を作成します。
**注意:** ステレオキャリブレーションを前提とする場合、「Cam1で悪かった画像」と「Cam2で悪かった画像」の**両方のインデックスを除外し、ペアを崩さない**ことが極めて重要です。

```python
# Cam1, Cam2 どちらかで悪かったインデックスをまとめる（重複を消す）
all_bad_indices = list(set(bad_indices1 + bad_indices2))
print(f"ペア全体から除外するインデックス一覧: {all_bad_indices}")

# クリーンなリストを用意
clean_objpoints = []
clean_imgpoints_cam1 = []
clean_imgpoints_cam2 = []

for i in range(len(objpoints_cam1)):
    if i not in all_bad_indices:
        clean_objpoints.append(objpoints_cam1[i])
        clean_imgpoints_cam1.append(imgpoints_cam1[i])
        clean_imgpoints_cam2.append(imgpoints_cam2[i])

print(f"除外後の有効な画像ペア数: {len(clean_objpoints)}")
```

## 5. キャリブレーションの再実行（再挑戦）

クリーンになった配列を使って `cv2.calibrateCamera` と `cv2.stereoCalibrate` を再度実行します。
ここで RMS（`ret1`, `ret2`, `ret_stereo`）が **1.0 px 未満（理想は 0.5 px 未満）** に劇的に下がっていれば、犯人探しのミッションは成功です。

```python
# グレースケール画像のサイズが必要 (gray_shapeは適宜ご自身の変数に置き換えてください)
gray_shape = (1920, 1080) # 例: 横x縦

# Cam1 の再キャリブレーション
ret1_clean, K1_clean, D1_clean, rvecs1_clean, tvecs1_clean = cv2.calibrateCamera(
    clean_objpoints, clean_imgpoints_cam1, gray_shape, None, None
)
print(f"Cam1 クリーン再計算時の RMS: {ret1_clean:.4f} px")

# Cam2 の再キャリブレーション
ret2_clean, K2_clean, D2_clean, rvecs2_clean, tvecs2_clean = cv2.calibrateCamera(
    clean_objpoints, clean_imgpoints_cam2, gray_shape, None, None
)
print(f"Cam2 クリーン再計算時の RMS: {ret2_clean:.4f} px")

# この後、K1_clean, D1_cleanなどを用いて cv2.stereoCalibrate に繋げます
```

import cv2

def get_video_info(path):
    cap = cv2.VideoCapture(path)
    if not cap.isOpened():
        print(f"Error: Could not open video file {path}")
        return {"fps": 0, "width": 0, "height": 0, "n_frames": 0, "duration_sec": 0}

    fps    = cap.get(cv2.CAP_PROP_FPS)
    width  = int(cap.get(cv2.CAP_PROP_FRAME_WIDTH))
    height = int(cap.get(cv2.CAP_PROP_FRAME_HEIGHT))
    n_frames = int(cap.get(cv2.CAP_PROP_FRAME_COUNT))
    duration = n_frames / fps if fps > 0 else 0
    cap.release()
    return {"fps": fps, "width": width, "height": height,
            "n_frames": n_frames, "duration_sec": duration}

#@title 動画ファイルの選択 { display-mode: "form" }
