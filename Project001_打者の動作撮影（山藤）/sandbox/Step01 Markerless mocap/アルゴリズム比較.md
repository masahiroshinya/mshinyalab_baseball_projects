作成日：2026-03-25

## 候補アルゴリズム一覧

### ヒト姿勢推定

| ライブラリ | 開発元 | キーポイント数 | 処理速度 | GPU要否 | 特徴 |
|-----------|-------|-------------|---------|---------|------|
| **MediaPipe Pose** | Google | 33点（全身） | 高速 | 不要（CPU可） | セットアップが簡単。スポーツ分析の使用例が多い |
| **YOLOv8-Pose** | Ultralytics | 17点（COCO形式） | 非常に高速 | 推奨（CPU可） | 物体検出と統合可能。バット追跡と同一フレームワークで扱える |
| **OpenPose** | CMU | 25点（全身） | 低速 | 推奨 | 多人数対応。学術評価基準が多く、論文比較に有利 |
| **MMPose** | OpenMMLab | モデル依存 | モデル依存 | 推奨 | 多数のモデルをサポート。設定が複雑。研究用途向け |

### バット追跡（物体検出）

| ライブラリ | ライセンス | 商用利用 | 特徴 |
|-----------|----------|---------|------|
| **YOLOv8** | AGPL-3.0 | **要有料ライセンス** | 姿勢推定モデルと同一フレームワーク。カスタムデータ学習が容易。Roboflow にバット検出用データセットあり |
| **RT-DETR (MMDetection)** | Apache 2.0 | **無料・自由** | MMPose と同じ OpenMMLab フレームワーク。姿勢推定と統一可能 |
| **OpenCV DNN** | Apache 2.0 | **無料・自由** | 追加インストール不要。軽量だが検出精度はやや低い |
| **ByteTrack / SORT** | MIT | **無料・自由** | 追跡（トラッキング）専用。別途検出器と組み合わせて使用 |

> **商用展開を将来考える場合**：MMPose（姿勢推定）＋ RT-DETR（バット検出）で Apache 2.0 に統一するのが安全です。研究・教育目的のみであれば YOLOv8 で姿勢推定・バット追跡の両方を統一する方が実装が簡単です。

## 精度・ライセンス比較

### 精度（COCO val2017 ベンチマーク）

姿勢推定の精度は一般に **AP（Average Precision）** で評価されます。値が高いほど精度が高く、100% が上限です。

| ライブラリ / モデル | AP (mAP50-95) | 評価データセット | 備考 |
|-------------------|--------------|----------------|------|
| **MediaPipe Pose Full** | ~65–67% | COCO val2017相当 | PCKh 指標では ~91%（異なる評価方法） |
| **YOLOv8n-Pose**（最小） | ~50% | COCO val2017 | 軽量・高速モデル |
| **YOLOv8s-Pose** | ~60% | COCO val2017 | 小型モデル |
| **YOLOv8m-Pose** | ~65% | COCO val2017 | 中型モデル |
| **YOLOv8l-Pose** | ~68% | COCO val2017 | 大型モデル |
| **YOLOv8x-Pose**（最大） | ~69% | COCO val2017 | 最高精度モデル |
| **OpenPose (Body_25)** | ~61% | COCO val2017 | 古い基準モデル |
| **MMPose / RTMPose-l** | ~76% | COCO val2017 | 高精度。モデル選択の幅が広い |

> **注意**：COCO ベンチマークは一般動作の評価であり、バッティング動作のような高速・特殊動作での精度は別途検証が必要です。

## 各アルゴリズムの詳細

### MediaPipe Pose（BlazePose）

- **インストール**：`pip install mediapipe`
- **キーポイント**：33点（顔・体幹・手先・足先）
- **入力**：静止画 / 動画どちらも対応
- **長所**
  - CPU 環境でもリアルタイム動作
  - ドキュメントと公式サンプルが豊富
  - 手・顔の詳細なキーポイントも取得可能
- **短所**
  - 1人のみ検出（多人数非対応）
  - バット等の物体検出機能はなし
  - モデルの差し替えが難しい

### YOLOv8-Pose

- **インストール**：`pip install ultralytics`
- **キーポイント**：17点（COCO形式：鼻・肩・肘・手首・腰・膝・足首など）
- **入力**：静止画 / 動画 / リアルタイムカメラ対応
- **長所**
  - 姿勢推定と物体検出（バット）を**同一フレームワーク**で統一できる
  - GPU があれば非常に高速、CPU でも動作可
  - カスタムデータでのファインチューニングが容易
  - コード数行で動作確認可能
- **短所**
  - キーポイントが 17点（手先・足先の詳細なし）
  - MediaPipe より細かい部位（指先など）は取れない

### OpenPose

- **インストール**：やや複雑（C++のビルドが必要な場合あり）
- **キーポイント**：25点（全身）
- **長所**
  - 多人数に対応
  - 学術論文での比較基準として多用される
- **短所**
  - GPU 推奨（CPU では非常に低速）
  - セットアップが重い
  - 2024年以降のメンテナンスが活発でない

---

## 推奨案

**YOLOv8-Pose を採用することを推奨します。**

理由：

1. **バット追跡との統一**：YOLO-Detect と YOLO-Pose を同一フレームワークで扱えるため、Step 3（バット追跡）・Step 4（同時トラッキング）への発展が最もスムーズ
2. **実装の簡単さ**：数行のコードで動作確認でき、学生が短期間で習得しやすい
3. **性能のバランス**：CPU 環境でも動作し、研究室の PC 環境を選ばない
4. **活発な開発**：Ultralytics によって継続的にメンテナンスされており、ドキュメントが充実

比較のために MediaPipe でも同じ画像を処理し、精度・使いやすさを確認することを推奨します。

## 採用方針（決定済み）

**当面は YOLOv8 で開発を進め、商用化の目処が立った時点で MMPose + RT-DETR に移行する。**

- 研究・技術開発フェーズ：**YOLOv8**（姿勢推定 + バット検出を統一。実装が簡単で学生も習得しやすい）
- 商用化フェーズ：**MMPose + RT-DETR**（Apache 2.0。ライセンスコストなしで商用展開可能）
