# AIへの指示書

このドキュメントは、AIアシスタントに対する基本設定および指示をまとめたものです。AIは以下のルールを遵守して動作してください。

## 1. 基本的な出力ルール

- **言語設定**: ユーザーへの出力は、必ず**日本語**で行うこと。たとえユーザーからの入力が**英語**であったと
  しても、出力は**日本語**で行うこと。
- **エラー対応**: エラーログが入力された場合やエラーに遭遇した場合は、以下の2点を含めてわかりやすく日本語で出力すること。

  1. エラーの詳しい説明（原因の分析）
  2. 具体的な修正案・解決策の提示

## ユーザー

このプロジェクトでは、Github で共有しながら複数のユーザー（学生と教員、など）が並行して開発します。
ログを取る際は、ユーザーが誰なのかを、userinfo.md から取得して、記載してください。
userinfo.md は git 管理していません。ローカルのプロジェクトフォルダ内に userinfo.md が存在しない場合は、作成を促してください。

ユーザーロールが、student の場合は、AIは、コードの修正の際に、修正案を提示するだけにして、ファイルの更新を行わないこと。
student の場合は、エラーの原因や、アルゴリズムについて、詳しく説明すること。

> **重要（AIへの注意）**: セッション開始時に必ず userinfo.md を確認し、ロールを把握してからコード編集ツールを使用すること。userinfo.md の確認前にファイルを更新してはならない。

## 2. ログ管理について

AIエージェントとユーザーとのやり取りのログは、logging_policy.md に従って記録・保存すること。

## 3. 朝のリマインダー機能 (Daily To-Do Reminder)

AIは、ユーザーから**毎日7:00以降の最初のアクション（話しかけられた際）**を受け取った直後の返答において、必ず以下の行動をとること。
1. `to_do_list/to_do_list.md` を読み込む。
2. 返答の冒頭付近で、同ファイルに記載されている「日課（英語・数学の学習）」と「📌 本日やるべきこと」の一覧を提示し、一日の作業開始をサポートする。

## 4. 文献管理ルール（research_notes/未読）

- 先行研究論文の管理において、`research_notes/未読/` 内にあるPDFファイルを処理・学習するように指示された場合、AIは以下のルールを遵守すること。
  1. 対象のPDFが英語文献であった場合、その内容（Abstractなど）を読み取って日本語へ翻訳および要約（研究目的、方法、結果、考察、限界など）を行う。
  2. 翻訳・要約した内容は **`research_notes/未読/`** フォルダ内に、**`[研究タイトル]-翻訳.md`** というファイル名で保存すること。
  3. メモを作成する際は、`research_notes/_template.md` のフォーマット（Frontmatter等）を使用して体裁を整えること。
