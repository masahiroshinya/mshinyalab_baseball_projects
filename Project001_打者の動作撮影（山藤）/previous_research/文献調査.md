# 文献調査フレームワーク

このドキュメントは、本プロジェクトにおいて先行研究や原著論文を調査・評価する際の標準的なフレームワーク（テンプレート）です。今後の文献調査は、この基準に基づいて実施・記録してください。

### ファイル・ディレクトリの保存ルール
* **調査リスト/まとめ**： リスト化して保存するファイルは `survey` フォルダ内に `yyyy-mm-dd-survey.md` の形式で保存します。
* **日々の調査ログ/個別記録**： 個別の調査や関連するログファイルは `logs` フォルダ内に保存します。
---

## 1. 調べる際に必要な情報（調査の前提）
文献調査を開始する前に、以下の項目を明確にしてから検索を行います。

*   **調査の目的**: 何のためにこの文献を探すのかを明記します。目的によって適切なアプローチが異なります。
    *   **分野の全体像を知りたい**: 過去の歴史や現在のトレンドを把握するため、総説（Review論文）や被引用数の多い主要な論文を探す。
    *   **研究目的・背景を調べたい**: 他の研究者がどのような課題を解決しようとしているか、論文の「Introduction（導入）」部分を中心に探す。
    *   **研究方法（手法・技術）について調べたい**: 使用機材、アルゴリズムの実装詳細、パラメータ設定など、具体的な技術的解決策を探す。
*   **対象となるシステム・環境**: （例：民生用アクションカメラ、複数台の同期、屋外での計測）
*   **求める成果物の種類**: 原著論文、総説（Review）、テクニカルレポート、実装コード（GitHub等）
*   **限界・制約**: （例：ソフトウェアのみで解決できる手法、高価な機材を必要としない手法）

## 2. キーワード検索の方法

効率的に質の高い学術論文・文献を見つけるための検索戦略です。

### 2-1. 使用データベース
*   **Google Scholar**: 幅広い分野の網羅的な検索（第一選択）
*   **PubMed**: バイオメカニクス、スポーツ医学系の論文
*   **IEEE Xplore / ACM Digital Library**: コンピュータビジョン、トラッキング、同期技術系
*   **arXiv / CVF (Computer Vision Foundation)**: 深層学習、最新のAI手法（OpenPose, YOLO等）

### 2-2. 検索クエリの組み立て（ブリアン検索）
*   **AND検索**: `Markerless AND Biomechanics` （両方の単語を含む）
*   **OR検索**: `"Action Camera" OR "GoPro" OR "OSMO"` （同義語や類似デバイスの網羅）
*   **フレーズ検索**: `"3D Reconstruction"` （複合語をそのままの語順で検索）
*   **除外検索**: `-marker-based` （不要なノイズを除外）

### 2-3. キーワード策定シート（例）
| 日本語キーワード | 英語キーワード（検索用） | 関連語・同義語（OR検索用） |
| :--- | :--- | :--- |
| マーカーレス | Markerless | Pose estimation, Deep learning, OpenPose |
| 動作分析 | Motion capture | Kinematics, Biomechanics, Motion analysis |
| 同期 | Synchronization | Cross-correlation, Time alignment, Multi-camera |
| 幾何学補正 | Calibration | Fisheye distortion, 3D reconstruction |

## 3. 文献の採点基準（100点満点）

発見した文献が本プロジェクトにどの程度寄与するかを、以下の5つの観点（各20点満点）で評価し、合計点で推薦スコアを算出します。

| 評価項目 | 評価の観点 | 配点 |
| :--- | :--- | :--- |
| **A. 関連性 (Relevance)** | プロジェクトの課題（マーカーレス、バット追跡等）に直接的に関わっているか？ | 20点 |
| **B. 実用性 (Applicability)** | 提案されている手法やコードが、現状の機材（アクションカメラ等）で再現・実装可能か？ | 20点 |
| **C. 信頼性 (Reliability)** | 査読付きの質の高い論文誌であるか？進矢研究室の学術的基準を満たすか？ | 20点 |
| **D. 独自性 (Novelty/Value)**| これまでの調査で得られていない新たな知見や、行き詰まりを打破するアイデアが含まれているか？ | 20点 |
| **E. 客観的指標 (Impact Metrics)**| 引用数（Citation）や閲覧数が多く、学術界で支持されている重要な文献であるか？ | 20点 |

*   **90〜100点**: プロジェクトの根幹に関わる、即座に深く読み込みチーム内で共有すべき【必読文献】
*   **70〜89点**: 実装の手がかりや評価指標の参考になるため、重要な章を抽出して読むべき【重要文献】
*   **50〜69点**: 直接的な応用は難しいが、背景知識や関連研究の引用元としてリストアップしておく【参考資料】
*   **50点未満**: 今回のプロジェクトのスコープからは外れる（記録のみ、または除外）

## 4. 論文まとめテンプレート（記入用）

実際に文献を読んだ際は、以下のフォーマットをコピーして記録します。

```markdown
### 【論文タイトル】
*   **著者・発表年**: (例: Nakano et al., 2020)
*   **掲載元**: (例: Frontiers in Sports and Active Living)
*   **URL/DOI**: 
*   **引用数 / 閲覧数**: (例: 引用 150件 / 閲覧 2,000回)
*   **推薦スコア**: [総合点]点 (A:点, B:点, C:点, D:点, E:点)
*   **分野・ジャンル**: (例: トラッキング, 幾何学精度, キャリブレーション, バイオメカニクス)

#### 1. 論文の概要 (Abstractの要約)
*   何を解決しようとした研究か？
*   どのような手法を用いているか？

#### 2. 本プロジェクトへの評価・適用方針 (Why is this important?)
*   この論文から何を取り入れるか？
*   私たちのチームが直面しているどの課題に対する解決策になるか？

#### 3. 技術的な詳細・注意点
*   （実装する上で必要になりそうなパラメータ、制約、計算量などがあれば記載）
```
