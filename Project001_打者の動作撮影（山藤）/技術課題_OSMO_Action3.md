# 技術課題：OSMO Action 3等アクションカメラでの撮影・解析

当プロジェクトでは、2〜4台のOSMO Action 3を使用し、1080p・240fpsの設定で撮影を行います。アクションカメラの特性および高フレームレートでの撮影に起因する特有の技術課題と、その対策（ベストプラクティス）を以下にまとめます。

## 1. レンズの歪み（ディストーション）への対策
アクションカメラは画角が非常に広く、周辺部に強い樽型歪み（フィッシュアイ効果）が発生します。
*   **課題**: 通常のピンホールカメラモデルを前提としたキャリブレーション（`cv2.calibrateCamera`）では、画像の端の方の歪みを正確に補正しきれず、3次元再構成での再投影誤差（RMS）が大きくなる原因になります。
*   **対策**:
    1.  キャリブレーション撮影時、**画面の中央だけでなく四隅の縁ギリギリまで**チェスボードを移動させて撮影する。
    2.  通常のキャリブレーションでRMSが下がらない場合、OpenCVの `cv2.CALIB_RATIONAL_MODEL` フラグを追加するか、魚眼カメラモデル（`cv2.fisheye.calibrate`）への移行を検討する。

## 2. 電子手ブレ補正は必ず「OFF」にする
OSMO Action 3などのカメラには強力な電子手ブレ補正（RockSteady等）が搭載されています。
*   **課題**: 電子手ブレ補正は、ジャイロセンサーの動きに合わせてリアルタイムで画像クロップの範囲をずらし、映像を安定させる機能です。これがオンになっていると、フレームごとにカメラの内部パラメータ（光軸の位置など）が変わるため、**固定カメラを前提とする幾何学的計算（キャリブレーションや三角測量）が完全に破綻**します。
*   **対策**: 撮影時は必ず電子手ブレ補正系の機能を**すべてOFF**にし、画角（FOV）は標準（WideやDewarpなど）に固定して撮影を行う。

## 3. 240fpsの高精度な時間同期
1080p @ 240fpsの設定は、高速で動くバットのモーションブレを防ぐのに最適です。しかし、同期についてはよりシビアになります。
*   **課題**: 240fpsでは1フレームあたり約4.17ミリ秒しかありません。現在用いている「音声（クロスコーレーション）による同期」は概算には非常に有効ですが、カメラ間の録音サンプリングレートの僅かなズレなどが原因で、最終的に1〜数フレームのズレが残る可能性があります。
*   **対策**: 動画内に**視覚的に明確なイベント**（例：フラッシュを焚く、手を叩いた瞬間の接触フレーム、バットにボールが当たった瞬間）を用意し、最終的な時間同期は画像を並べて「1フレーム単位（1/240秒）」で目視・手動微調整を行う。

## 4. 膨大なデータ量と計算コスト（事前トリミング）
*   **課題**: 240fps映像はデータ量が膨大です（10秒で2400フレーム）。これをそのままGoogle Colabにアップロードし、全フレームに対してYOLO推論や3次元再構成をかけると、処理時間やメモリ消費が非現実的になります。
*   **対策**: 解析に必要な**「スイング前後の1〜2秒間（240〜480フレーム）」**だけを正確に切り出してから（トリミングしてから）、YOLO推論などの重い処理パイプラインに入力するフローを徹底する。

## 5. シャッタースピードと明るさ
*   **課題**: 240fpsで撮影するということは、1フレームあたりの最大露光時間が1/240秒未満になることを意味します。屋内など光量が不足する環境では、映像が極端に暗くなったり、ノイズが多くなってYOLOの検出精度が低下します。
*   **対策**: 屋外の直射日光下、もしくは屋内の場合は非常に強い照明を用意し、ISO感度が上がりすぎないようにする。
