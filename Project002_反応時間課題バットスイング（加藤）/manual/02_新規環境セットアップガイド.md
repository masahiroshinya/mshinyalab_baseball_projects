# 新しい環境でのセットアップ手順

新しいパソコンや別の作業環境で、このプロジェクト（Project002）を開始するための手順です。

## 1. プロジェクトをダウンロードする（Clone）

作業をしたい場所（フォルダ）に移動し、GitHubからデータをまるごとダウンロードします。

```bash
git clone https://github.com/masahiroshinya/mshinyalab_baseball_projects.git
```

ダウンロードが完了すると、`mshinyalab_baseball_projects` というフォルダが作成されます。その中の `Project002_反応時間課題バットスイング（加藤）` フォルダを開いて作業を開始します。

## 2. ユーザー情報の設定（userinfo.md の作成）

このプロジェクトでは、AI（私）が「誰が作業しているか」を把握するために、Gitで管理されないローカル専用のファイル `userinfo.md` を使用します。

プロジェクトのフォルダ直下（`Project002_反応時間課題バットスイング（加藤）/` の中）に、以下の内容で `userinfo.md` というファイルを作成してください。

```markdown
# ユーザー情報
- **氏名**: [あなたの氏名]
- **ロール**: student（または教員・管理者など）
```

> [!IMPORTANT]
> このファイルがないと、AIがあなたのロール（権限）を把握できず、一部の自動作業が制限される場合があります。必ず最初に作成してください。

## 3. 進捗状況の確認（To-Do List）

作業を開始する前に、`to_do_list/` フォルダの中身を確認してください。

- `01_today.md`: 今日やるべきことが書かれています。
- `04_completed.md`: すでに終わった作業が書かれています。

## 4. 日々の作業サイクルへ

セットアップが完了したら、あとは「基本の5ステップ」に従って作業を進めるだけです。

1. `git pull origin main --rebase` （最新状態にする）
2. 作業する
3. `git add .` （変更をリストに乗せる）
4. `git commit -m "メッセージ"` （セーブする）
5. `git push origin main` （GitHubへ送る）
