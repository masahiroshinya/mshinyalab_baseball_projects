## STEP 1: 実験条件の整理（設計の前提確認）

実験の「何を制御したいか」を言語化する段階です。コードを書く前に、紙またはメモで整理します。

### 1-1. 条件の種類を決める

以下の**3条件**で実施する。

| 条件ラベル（案） | 内容 | LED刺激 |
|----------------|------|---------|
| `no_stimulus` | 被験者が自分のペースでバットを振る | なし |
| `simple_reaction` | 1種類のLEDが点灯したら素早くバットを振る（常にGo） | 1種類（Goのみ） |
| `go_nogo` | GoのLED → 振る／No-GoのLED → 振らない | 2種類（Go + No-Go） |

> **2026-04-27 確認済み：** 条件は上記3種類（刺激なし・単純反応・Go-No Go）

**確定パラメータ（2026-04-27 先行研究調査より）：**

| パラメータ | 確定値 | 根拠 |
|-----------|--------|------|
| 単純反応 総試行数 | **40〜60 試行** | 疲労と統計検出力のバランス（Kida 2005 etc.） |
| Go-No Go 総試行数 | **60〜100 試行** | NoGo抑制検出に必要な試行数 |
| バットスイングのみ条件 | **20〜30 試行** | 全身運動のため身体疲労を考慮 |
| Go : No-Go 比率 | **70 : 30** | Young(2018), Wessel(2018)。生態学的妥当性も高い |
| 練習試行数（単純反応） | **10 試行** | Kida(2005)準拠 |
| 練習試行数（Go-NoGo） | **15〜20 試行** | 色弁別ルール習熟に必要 |

- [x] 1条件あたりの試行数（上記参照）
- [x] `go_nogo` 条件の Go:No-Go 比率 = **70:30**
- [x] 練習試行あり（条件切り替えごとに実施）
- [x] Go/No-Go区別方法 = **案B（RGB LED色違い）** → STEP 1-3 参照

### 1-2. ブロック構造を決める

| 設計案 | 説明 |
|--------|------|
| 案A: 条件ごとにブロックを分ける | ブロック1=self_paced, ブロック2=reactive |
| 案B: ブロック内でランダムに混在させる | 1ブロック内に両条件をランダム配置 |
| 案C: 条件ブロックを複数回繰り返す | self → reactive → self → reactive ... |

**確定パラメータ（2026-04-27 先行研究調査より）：**

- [x] 採用案：**案A（条件ごとにブロックを分ける）**
- [x] ブロック内の試行数：**20〜30 試行 × 複数セット**（Go-NoGoは2〜3セット分割）
- [x] 条件提示順：**ラテン方格法による6通りカウンターバランス**

| 推奨順序例 | 1番目 | 2番目 | 3番目 |
|-----------|-------|-------|-------|
| 順序A | 刺激なし | 単純反応 | Go-No Go |
| 順序B | 刺激なし | Go-No Go | 単純反応 |
| 順序C | 単純反応 | 刺激なし | Go-No Go |
| 順序D | 単純反応 | Go-No Go | 刺激なし |
| 順序E | Go-No Go | 刺激なし | 単純反応 |
| 順序F | Go-No Go | 単純反応 | 刺激なし |

> **根拠（Kida 2005 + 疲労対策）：** 条件混在（ミックスド）デザインはバットスイングの大動作に不向き。条件ブロック化でルール切り替え認知負荷を排除し、解析も容易になる。

### 1-3. 使用するLEDチャンネルを確認する

NI DAQボードのどのチャンネルにどのLEDが接続されているかを確認します。

> **2026-04-27 確認済み：** RGB LED 1個を使用。R・G・B の各端子を別々のAOチャンネルに接続する構成（案B-2）。

**RGB LED の配線イメージ：**

```
NI DAQ AO チャンネル
  ao? ──→ R 端子（赤）
  ao? ──→ G 端子（緑）
  ao? ──→ B 端子（青）
  GND ──→ 共通カソード（or VCC → 共通アノード）
```

**確認済み事項：**
- [x] Go刺激の色 = **緑**、No-Go刺激の色 = **赤**、単純反応刺激の色 = **白**
- [x] DAQ出力電圧は **5V 固定**（色の違いはどのチャンネルをONにするかで実現。保護抵抗で電流制限）
- [x] CSVの色管理方式 = **色ラベル案**（`stimulus_color` 列に `"white"` / `"green"` / `"red"` / `"none"` を記載し、MATLABスクリプト側でチャンネルに変換）

**残りの確認事項：**
- [ ] R・G・B の各端子は、それぞれ何番のAOチャンネルに接続するか？（例：ao0=R, ao1=G, ao2=B）← 後回し可
- [ ] Qualisys同期用チャンネルは何番か？（例：ao3）← 後回し可

**色 → チャンネル対応表（チャンネル番号確定後に埋める）：**

| 色 | 対応チャンネル | 点灯電圧 | 使用条件 |
|----|--------------|---------|---------|
| 白（R+G+B 同時点灯） | ao?, ao?, ao? | 各 5V | `simple_reaction` |
| 緑（G のみ点灯） | ao? | 5V | `go_nogo` の Go 試行 |
| 赤（R のみ点灯） | ao? | 5V | `go_nogo` の No-Go 試行 |
| なし（全消灯） | — | 0V | `no_stimulus` |

> **白色の注意：** R・G・B を同時に 5V で点灯すると白く見えますが、実際の白さはLEDの個体差に依存します。白LEDとは別物であることに留意してください。

### 1-4. タイミングパラメータを決める

| パラメータ | 意味 | 単位 |
|------------|------|------|
| `isi_sec` | 前試行の終了から次のLED点灯開始までの間隔（刺激間隔） | 秒 |
| `jitter_sec` | ISIに加えるランダムゆらぎ幅（予測困難性のため） | 秒 |
| `led_duration_sec` | LEDを点灯させ続ける時間 | 秒 |
| `warning_sec` | （任意）スタート合図から刺激までの予告時間 | 秒 |

**確定パラメータ（2026-04-27 先行研究調査より）：**

| パラメータ | 確定値 | 根拠 |
|-----------|--------|------|
| `isi_sec` 基準値 | **3〜6秒ランダム（平均 4.5秒）** | Kida(2005) 3-5秒 + バットスイング復帰時間考慮 |
| `jitter_sec` | **平均ISIの ±25〜33%**（例：±1〜1.5秒） | Verleger(2008)、Vaurio(2009) |
| `led_duration_sec` | **0.150秒（150 ms）** | Go-NoGo色弁別に必要な視認時間、RT解析の簡素化 |
| `voltage_on` | **5.0 V** | NI DAQ 固定（保護抵抗で電流制限） |
| RT除外 下限 | **100 ms** | 生理学的下限（予期反応排除） |
| RT除外 上限 | **1500 ms + 平均±3SD補助** | 全身動作の遅延考慮 |

- [x] `isi_sec` = 3〜6秒ランダム（CSVには `isi_min_sec=3.0`, `isi_max_sec=6.0` で管理）
- [x] `jitter_sec` = ISI内のランダムばらつきとして実現（一様分布 3〜6秒）
- [x] `led_duration_sec` = 0.150（秒）
- [x] `voltage_on` = 5.0 V

## STEP 2: CSVのカラム設計

STEP 1 の整理を踏まえて、CSVの「列の定義」を決定します。

### 2-1. 基本カラムのリストアップ

**確定版（2026-04-27）**

| カラム名 | 型 | 説明 | 例 |
|----------|----|------|----|
| `trial_id` | int | 通し試行番号（1始まり） | 1, 2, 3, ... |
| `block_id` | int | ブロック番号（1始まり） | 1, 1, 2, ... |
| `trial_in_block` | int | ブロック内試行番号（1始まり） | 1, 2, 1, ... |
| `condition` | str | 実験条件 | `no_stimulus`, `simple_reaction`, `go_nogo` |
| `trial_type` | str | 試行種別 | `practice`, `main` |
| `stimulus_color` | str | LED色ラベル | `none`, `white`, `green`, `red` |
| `response_required` | str | 期待される反応（正誤判定の基準） | `n/a`, `go`, `nogo` |
| `isi_sec` | float | 刺激間隔（乱数生成済み、3.0〜6.0秒） | 4.231 |
| `led_duration_sec` | float | LED点灯継続時間（秒） | 0.150 |
| `voltage_on` | float | LED点灯電圧（V） | 5.0 |
| `notes` | str | 備考（任意） | `""` |

**設計上の主要決定事項：**
- `stimulus_color` で色ラベル管理（MATLABスクリプトがチャンネルに変換）
- `isi_sec` = 3.0〜6.0秒の一様乱数（事前生成値をそのまま格納）
- `jitter_sec`・`onset_sec`・`sync_channel` は廃止（冗長または設定セクションで管理）
- `trial_type` で練習/本試行を明示的に区別
- `response_required` で解析時の正誤自動判定を可能にする

- [x] カラム定義確定（2026-04-27）

MATLABがCSVを読み込んだ後に「おかしなデータ」を検出するためのチェック項目を決めます。

### 3-1. チェック項目リスト

| チェック項目 | 内容 | 対応方法 |

## STEP 4: CSVを自動生成するMATLABスクリプトの設計

手動でCSVを作ると試行数が多い場合に現実的でないため、MATLABスクリプトで自動生成します。

### 4-1. スクリプトの入力パラメータ

スクリプトの冒頭に以下を「設定セクション」として記述します。

```matlab
%% ===== 設定セクション =====
n_blocks          = 4;        % ブロック数
n_trials_per_block = 20;      % 1ブロックあたりの試行数
conditions        = {'self_paced', 'reactive'};  % 条件リスト
led_channels      = {'none', 'ao0'};             % 条件に対応するLEDチャンネル
isi_base_sec      = 2.0;      % ISI基準値 [秒]
jitter_min_sec    = 0.0;      % ジッター最小値 [秒]
jitter_max_sec    = 1.0;      % ジッター最大値 [秒]
led_duration_sec  = 0.3;      % LED点灯時間 [秒]
voltage_on        = 5.0;      % 点灯電圧 [V]
sync_channel      = 'ao2';    % Qualisys同期チャンネル
output_filename   = 'protocol_YYYYMMDD.csv';  % 出力ファイル名
```

### 4-2. スクリプトの処理フロー

```
[開始]
  ↓
設定パラメータの読み込み
  ↓
乱数シードの設定（再現性のため rng(固定値) を使用）
  ↓
全試行のデータをループで生成
  ├─ ブロックループ（block_id = 1〜n_blocks）
  │    └─ 試行ループ（trial_in_block = 1〜n_trials_per_block）
  │          ├─ 条件をランダム選択（またはブロック割り当て）
  │          ├─ jitter_sec を乱数で生成
  │          ├─ onset_sec = isi_base_sec + jitter_sec を計算
  │          └─ 1行分のデータを構造体 or テーブルに追加
  ↓
trial_id を通し番号で付与
  ↓
バリデーション実行（STEP 3 の項目）
  ↓
CSVファイルへ書き出し（writetable 使用）
  ↓
確認メッセージを表示（総試行数、条件ごとの試行数）
[終了]
```
