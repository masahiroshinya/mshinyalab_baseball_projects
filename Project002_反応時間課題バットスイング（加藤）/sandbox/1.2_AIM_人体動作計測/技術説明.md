# 1.2. Qualisys の AIM を用いた人体動作計測

**作成日**: 2026-04-21  
**作成者**: mshinyalab (AI支援)

---

## 概要

Qualisys（モーションキャプチャシステム）の **AIM（Automatic Identification of Markers）** 機能を用いて、人体の動作を計測します。

AIM は、身体各部位に貼付した反射マーカーの3次元座標を追跡し、関節角度・セグメントの運動学データを自動的に計算する機能です。マーカーが一時的に隠れても、モデルから位置を推定・補間できることが特徴です。

---

## 全体の流れ

```
① マーカーの貼付
　　↓
② QTM 起動 → プレビュー開始
　　↓
③ 3D ビューでマーカーの表示を確認
　　↓
④ キャプチャ設定の確認（フレームレート等）
　　↓
⑤ QTM をトリガ待機状態にする
　　↓
⑥ MATLAB からカウントダウン付きトリガ送信 → キャプチャ開始・動作実施
　　↓
⑦ Trajectory Editor でラベリング
　　↓
⑧ ギャップ（途切れ）の確認と補間
　　↓
⑨ 3D ビューで動作を再生確認
　　↓
⑩ データのエクスポート
```

---

## ① マーカーの貼付

### テスト段階のマーカーセット（片側5点 × 両側 = 計10点）

| 解剖学的ランドマーク | 日本語 | 右側ラベル | 左側ラベル |
|---------------------|--------|-----------|-----------|
| Greater Trochanter  | 大転子 | `RGTRO` | `LGTRO` |
| Fibular Head        | 腓骨頭 | `RFIBH` | `LFIBH` |
| Lateral Malleolus   | 外果   | `RLMAL` | `LLMAL` |
| Calcaneus           | 踵骨   | `RCAL`  | `LCAL`  |
| 1st Metatarsal Head | 第1中足骨骨頭 | `R1MH` | `L1MH` |

### 命名規則

- `R` = Right（右）、`L` = Left（左）
- 以降の略語は解剖学名の頭文字に基づく
- 今後 AIM モデルを組む際は、モデルが要求するラベル名に変更が必要になる場合がある

### 貼付時の注意

- マーカーが剥がれないよう、肌にしっかり密着させる
- マーカーの反射面が上向き（カメラに向く方向）になっていることを確認する
- 左右の貼付位置が対称になっているか確認する

---

## ② QTM 起動 → プレビュー開始

1. QTM を起動し、対象プロジェクトを開く
2. **Capture → Preview** を選択してプレビューを開始する

---

## ③ 3D ビューでマーカーの表示を確認

1. **View → 3D** に切り替える
2. 貼付した10点のマーカーが白い点として表示されていることを確認する
3. マーカーが点滅・消えている場合は以下を調整する

   | 症状 | 対処法 |
   |------|--------|
   | 一部のマーカーが映らない | 立ち位置・マーカー向きを調整する |
   | 点滅する | カメラ台数・配置を見直す |
   | 2点が1点として認識される | マーカー間の距離を確認する |

---

## ④ キャプチャ設定の確認

**Project → Project Options → Capture** で以下を確認する。

| 設定項目 | 推奨値（テスト時） |
|----------|------------------|
| Capture frequency（フレームレート） | 100 Hz |
| Capture time（計測時間） | 5〜10秒 |
| Pre-trigger | OFF（テスト時） |

---

## ⑤ QTM をトリガ待機状態にする

1. **Capture → Capture** を選択する
2. Start Capture ダイアログで **Start** ボタンを押す
3. ステータスバーに **"Waiting for trigger"** と表示されることを確認する

> この表示が出る前に MATLAB からトリガを送るとエラーになるため、必ず確認してから次に進む。

---

## ⑥ MATLAB からトリガ送信 → キャプチャ・動作実施（一人作業の場合）

### スクリプトの実行手順

1. MATLAB のカレントフォルダを `sandbox/1.2_AIM_人体動作計測` に設定する
2. `2026-0421_01_practice_AIM_capture.m` を実行する
3. コマンドウィンドウの案内に従い **Enter** を押す
4. カウントダウン（デフォルト5秒）が表示される間に計測ポジションへ移動・静止する
5. 自動でトリガが送信され、QTM のキャプチャが開始される
6. `trialInterval` 秒（デフォルト5秒）後に MATLAB の計測待機が終了する
7. 2回目以降は QTM で再度 **Capture → Start** を押してから Enter を押す

### スクリプトの主要パラメータ

| パラメータ | 初期値 | 説明 |
|-----------|--------|------|
| `nTrials` | 3 | 計測するトライアル数 |
| `trialInterval` | 5（秒） | 1トライアルの計測時間 |
| `countdownSec` | 5（秒） | トリガ送信前のカウントダウン時間 |

> **カウントダウンが表示されないトラブル**：`2.2_LED点灯制御` フォルダ内の古いファイルを実行している可能性がある。カレントフォルダを確認すること。

---

## ⑦ Trajectory Editor でラベリング

キャプチャ完了後、一人作業のため静止キャプチャデータを用いてラベリングを行う。

### Trajectory Editor を開く

**View → Trajectory Editor** を選択する。タイムライン上に各マーカーの帯（トラジェクトリ）が表示される。

### ラベルのつけ方

1. タイムライン上のマーカーの帯を **右クリック**する
2. **Label** を選択する
3. マーカー名を入力する（上記マーカーセット表を参照）
4. **Enter** で確定する
5. 全10点に対して繰り返す

> ラベルを一度つけると、次のキャプチャでも同じラベルが引き継がれる。

---

## ⑧ ギャップ（途切れ）の確認と補間

### ギャップとは

動作中にマーカーがカメラの死角に入ると、トラジェクトリが途切れる現象。Trajectory Editor のタイムライン上で帯が途切れている箇所として確認できる。

### 補間手順

1. ギャップのある箇所（帯の途切れ）を選択する
2. 右クリック → **Fill Gap** を選択する
3. 補間方法を選ぶ（短いギャップは **Polynomial** が一般的）
4. **OK** で実行する

> ギャップが1秒以上の場合は補間精度が低下するため、再計測を検討する。

---

## ⑨ 3D ビューで動作を再生確認

1. **View → 3D** に切り替える
2. タイムラインの **再生ボタン** を押す
3. 全マーカーが滑らかに・連続して動いていることを確認する
4. 不自然なジャンプ・スパイクがないかチェックする

---

## ⑩ データのエクスポート

| 形式 | 用途 |
|------|------|

## ボーン（マーカー間の線）の作成方法

ラベリング完了後、マーカー間を線で繋いで木構造（スケルトン）を視覚化できる。

### 正しい手順：Edit → Bone → Create bone（ショートカット：B キー）

1. 3D ビュー上で **1点目のマーカーをクリック**して選択する
2. **Shift を押しながら2点目をクリック**して2点選択する（右側トラジェクトリ一覧から選択しても可）
3. キーボードの **B キー**を押す → 2点間に線（Bone）が作成される
4. 以下の組み合わせに対して繰り返す

| Bone | 始点 | 終点 |
|------|------|------|
| 右大腿 | `RGTRO` | `RKnee` |
| 右下腿 | `RKnee` | `RAnkle` |
| 右足（つま先） | `RAnkle` | `RToe` |
| 右足（踵） | `RAnkle` | `RHeel` |
| 左大腿 | `LGTRO` | `LKnee` |
| 左下腿 | `LKnee` | `LAnkle` |
| 左足（つま先） | `LAnkle` | `LToe` |
| 左足（踵） | `LAnkle` | `LHeel` |

### 試したが機能しなかった方法（参考）

| 試した方法 | 結果 | 理由 |
|-----------|------|------|
| Skeleton メニュー | 定義済みモデル（Sports・Animation・Traqr VR）しか選択できず、カスタムモデルは作成不可 | QTM の Skeleton 機能は既存テンプレートを使う仕様 |
| AIM → Generate Model | "Failed to generate model - AIM model can not be loaded" エラー | Generate Model は既存の `.qam` テンプレートファイルを必要とする。ファイルが存在しない場合は実行不可 |

## トラブルシューティング

| 症状 | 原因の可能性 | 対処法 |
|------|------------|--------|
| カウントダウンが表示されない | 古いファイルを実行している | カレントフォルダを `1.2_AIM_人体動作計測` に変更する |
| プレビューにマーカーが映らない | 画角外・マーカーの向き不良 | 立ち位置・マーカー向きを調整する |
| トラジェクトリにギャップがある | 動作中に死角に入った | カメラ配置を変更する、または Fill Gap で補間する |
| ラベルが保存されない | Enter で確定していない | ラベル入力後に必ず Enter キーを押す |
| 2点が1点として認識される | マーカーが近すぎる | マーカー間の距離を確保する |
| "Waiting for trigger" が出ない | Capture → Start の操作が未実施 | QTM 側で Start ボタンを押してから Enter を押す |
| AIM Generate Model がエラーになる | `.qam` テンプレートファイルが存在しない | Skeleton メニューは使わず、Edit → Bone → Create bone（B キー）でボーンを手動作成する |
