# 文献整理ルール (Instruction for Previous Research)

このフォルダ（`Previous Research`）に新しく文献（論文等）を追加する際は、以下の統一ルールに従って整理・配置を行ってください。AI（Antigravityなど）に作業を依頼する際にも、このルールが基準となります。

## 1. 論文ごとにサブフォルダを作成し、連番（通し番号）をつける
新しい文献を追加する際は、必ず「プロジェクトに集めた（追加した）順番」がわかるように、連番を振った専用のフォルダを作成します。
- フォルダ名の命名規則: `[通し番号]_[第一著者名]_et_al_[発行年]` 
- 例1: `01_Kida_et_al_2005`
- 例2: `02_Nasu_et_al_2020`

## 2. ファイルの順序を固定する（01_, 02_, 03_ の付与）
各サブフォルダ内において、「1. 本文」「2. 和訳」「3. ノート」が常にこの順序でエディタ上に並ぶよう、ファイル名の先頭にナンバリング（プレフィックス）を追加します。

**構成イメージ：**
1. `01_<原著論文のファイル名>.pdf`
2. `02_<和訳論文のファイル名>.pdf` （※和訳が存在する場合のみ）
3. `03_notes.md` （メモ帳ファイル）

## 3. notes.md のテンプレート
新規に `03_notes.md` を作成する場合は、以下のテンプレートをコピーして使用してください。

```markdown
# 論文メモ: [第一著者名] et al. ([発行年])

## 文献情報
- **タイトル**: [論文のタイトル]
- **著者**: [著者一覧]
- **発行年**: [発行年]

## 概要
(論文のアブストラクトや全文をもとに、研究の背景・目的・実験方法・主要結果・結論を日本語でまとめてください)

## 大事な部分・要点
(その論文が主に述べていること・主張・結論を簡潔な箇条書きで示してください)
- 
- 

## 自身のプロジェクトへの応用・検証したいこと
- 
```

## 4. 論文追加時に概要を必ず記述する
新規に `03_notes.md` を作成した際は、アブストラクト・和訳・本文等をもとに、**概要セクションに日本語で論文の概要を記述**してください。

- 概要には「背景・目的」「実験・方法」「主要な結果」「結論・意義」を含めることを基本とします
- AIに作業を依頼する場合は、アブストラクト（和訳）を提供するか、その旨を指示してください
- 概要が未記入のまま（テンプレートのプレースホルダー文言のまま）にならないよう注意してください

## 5. 「大事な部分・要点」には論文の主要な主張を簡潔に示す
`## 大事な部分・要点` セクションには、**その論文が主に述べていること（主張・発見・結論）を簡潔な箇条書きで記述**してください。

- 論文のキーメッセージが一目でわかる内容にまとめます
- 詳細な手続きや数値の羅列ではなく、「何がわかったか・何を主張しているか」を中心に記述します
- 各項目は太字（`**〜**`）で主張のキーワードを示したうえで、その補足を続けると読みやすくなります
- AIに作業を依頼する場合は、アブストラクトや結果・考察部分を提供するか、その旨を指示してください
